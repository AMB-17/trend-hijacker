================================================================================
DATABASE SCHEMA CHANGES
================================================================================

10 NEW TABLES:

1. alert_configs - User alert preferences
   - emailEnabled, slackEnabled, webhookEnabled
   - frequency (REAL_TIME, DAILY_DIGEST, WEEKLY_DIGEST)
   - digestHourOfDay, digestDayOfWeek for scheduling
   - mobileNotifications, inAppNotifications

2. alert_rules - Alert trigger conditions
   - name, description
   - minOpportunityScore, minVelocityThreshold
   - keywords[], stages[]
   - onlyNewTrends flag

3. alert_history - Log of all sent alerts
   - subject, message, channel
   - status (PENDING, SENT, FAILED, DELIVERED, BOUNCED)
   - deliveredAt, failureReason
   - retryCount, nextRetryAt (for exponential backoff)

4. alert_deliveries - Granular delivery tracking
   - historyId reference
   - channel, recipient
   - statusCode, statusMessage
   - externalId from email/Slack service

5. workspaces - Team containers
   - ownerId, name, description
   - isPrivate, stripeCustomerId

6. workspace_members - User membership
   - workspaceId, userId
   - role (ADMIN, EDITOR, MEMBER, VIEWER)
   - joinedAt, invitedAt, invitedBy

7. collections - Trend groupings
   - workspaceId, creatorId
   - name, description
   - isPublic, shareToken
   - itemCount (cached)

8. collection_items - Trends in collection
   - collectionId, trendId
   - trendSnapshot (trend data at time of save)
   - notes, tags[]

9. collection_comments - Collaborative notes
   - collectionId, userId
   - text (comment body)

10. workspace_activity_log - Audit trail
    - workspaceId, collectionId
    - action (15 action types)
    - description, actorId
    - targetId, targetType, metadata

5 NEW ENUMS:
  - AlertFrequency
  - AlertDeliveryStatus
  - WorkspaceRole
  - ActivityAction
  - (5th implied in schema)
